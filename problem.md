## Overview

Shell startup hangs indefinitely on every new zsh session. The hang occurs inside `compinit` / `compdump`, specifically during a filesystem glob operation. This report documents the full debugging session, all findings, all commands run, and their outputs — so that an AI coding agent can understand the full context and implement a fix.

**This is the hanging line.**

### Step 9: Identify the exact hanging code in `compdump`

```zsh
sed -n '105,120p' /nix/store/.../functions/compdump
```

Output:
```zsh
# Autoloads: look for all defined functions beginning with `_'
_d_als=($^fpath/(${(o~j.|.)$(typeset +fm '_*')})(N:t))
```

**This is the root cause.** This glob:
1. Gets all function names matching `_*` via `typeset +fm '_*'`
2. Builds a massive alternation pattern joining them all with `|`
3. Tries to match that pattern against every file in every `$fpath` directory
4. This is an **O(n²) operation** — n function names × n files per directory × number of fpath directories

---

### Step 10: Confirm `typeset +fm '_*'` alone is fast

```zsh
time zsh -df -c 'typeset +fm "_*"; echo done'
# done — 0.018s (instant, no functions loaded)
```

Fast when no functions are loaded. The problem is the **combination** of many loaded functions + the glob.

---

### Step 11: Reproduce the hang with functions loaded

```zsh
time zsh -df -c '
zmodload zsh/complete
zmodload zsh/complist
zmodload zsh/computil
autoload -Uz compinit
autoload -Uz ${fpath[1]}/_*(.:t)
echo "loaded $(typeset +fm "_*" | wc -l) functions"
_d_als=($^fpath/(${(o~j.|.)$(typeset +fm "_*")})(N:t))
echo "got ${#_d_als} entries"
echo done
'
# loaded 967 functions
# → HANGS after printing "967 functions"
```

**Confirmed: the compdump glob hangs when ~967+ functions are loaded.** This is the exact O(n²) bottleneck.


**Critical finding:** The dump filenames contain **PIDs** (31587, 31684, etc.), not a stable name. This means:
- A new unique dump file is created every shell session
- The existing dump is **never reused**
- `compdump` runs the expensive O(n²) glob on **every single shell startup**
- Something is calling `compinit -d ~/.zcompdump.$$` (using `$$` = PID) somewhere in the config

### Step 18: How fpath gets populated

Home-manager generates this loop in the zshrc:

```zsh
for profile in ${(z)NIX_PROFILES}; do
  fpath+=($profile/share/zsh/site-functions $profile/share/zsh/$ZSH_VERSION/functions $profile/share/zsh/vendor-completions)
done
```

`NIX_PROFILES` contains multiple profiles:
- `/Users/lluchkaa/.nix-profile`
- `/etc/profiles/per-user/lluchkaa`
- `/run/current-system/sw`
- `/nix/var/nix/profiles/default`

Each adds a `share/zsh/$ZSH_VERSION/functions` entry. Since multiple profiles may contain symlinks to the same zsh store path, the `5.9/functions` directory ends up in fpath multiple times.
